# CONTEXTO ACTUALIZADO DEL PROYECTO — ESTADO REAL DEL SISTEMA (UX/UI + FUNCIONALIDAD + ARQUITECTURA)

## STACK Y ENFOQUE GENERAL

El sistema es una plataforma modular de gestión operativa basada en:

* Sistema de tickets avanzado
* Gestión de inventario y activos
* Gestión administrativa interna
* Motor SLA dinámico
* Roles globales y por módulo
* Arquitectura modular configurable
* Dashboard operativo multirol
* Reportería y métricas en tiempo real
* Sistema de solicitudes internas administrativas

El enfoque principal del sistema es:

* Escalabilidad
* Bajo número de clics
* UX limpia
* Navegación rápida
* Separación entre administración y operación
* Configuración dinámica sin tocar código
* Estructura profesional tipo SaaS empresarial

# CALENDARIO GLOBAL DEL SISTEMA

El calendario NO será una página aislada simple.

Será un sistema operativo central conectado a:

* Roles
* Módulos
* Tareas
* Eventos
* Actividad operativa
* SLA
* Disponibilidad

## Decisión UX/UI

El calendario NO debe vivir dentro del sidebar.

Debe integrarse en el header superior mediante:

* Botón/icono contextual
* Dropdown expandible
* Modal lateral
* Vista rápida

Porque:

* Todos los roles lo usan
* Es transversal al sistema
* No pertenece únicamente al área administrativa
* Reduce saturación del sidebar

---

# COMPORTAMIENTO SEGÚN ROL

El calendario cambia dinámicamente según:

* Rol global
* Rol dentro del módulo activo
* Permisos específicos

## Ejemplo

Un técnico:

* Ve tareas técnicas
* No puede crear eventos globales

Un admin de módulo:

* Puede crear tareas para su módulo

Un superadmin:

* Ve todo
* Filtra por módulo
* Filtra por usuario
* Ve métricas cruzadas

modulo que le corresponde, recordar que se pueden seleccionar muchos a la

# INTEGRACIÓN DE VIDEOLLAMADAS Y COLABORACIÓN EN TICKETS

## Objetivo

El sistema deberá permitir colaboración en tiempo real dentro de los tickets, especialmente en:

* Casos complejos
* Escalamientos
* Soporte cooperativo
* Validaciones técnicas
* Atención directa al usuario

La funcionalidad debe integrarse principalmente dentro del módulo:

Path:
`/modules/helpdesk`

---

# VIDEOLLAMADAS DENTRO DEL SISTEMA

## Integración prevista

Se planea integrar servicios de videollamada como:

* Google Meet API
* Google Calendar API
* Alternativas futuras desacopladas

La implementación debe diseñarse desde el inicio de forma modular para permitir reemplazar el proveedor posteriormente.

---

# USOS DE LAS VIDEOLLAMADAS

## 1. Colaboración entre técnicos

Un técnico podrá:

* Solicitar apoyo de otro técnico
* Crear una videollamada colaborativa
* Compartir contexto del ticket
* Resolver incidentes conjuntamente

Especialmente útil para:

* Tickets críticos
* Casos especializados
* Reprocesos
* Escalamientos

---

## 2. Atención directa al usuario (evaluación futura)

Opcionalmente, el sistema podrá permitir:

* Videollamadas entre técnico y usuario final

Pero esto debe evaluarse cuidadosamente porque:

* Puede aumentar carga operativa
* Puede romper flujo asincrónico del sistema
* No todos los tickets requieren contacto directo

Por ahora, debe diseñarse como:

* Funcionalidad opcional
* Activable por módulo o política
* Controlada por permisos

---

# FUNCIONAMIENTO ESPERADO

Dentro del ticket se podrá:

* Crear reunión
* Generar enlace automáticamente
* Invitar participantes
* Registrar quién participó
* Registrar fecha y duración
* Asociar la reunión al historial operativo del ticket

---

# RESTRICCIONES Y PERMISOS

Las videollamadas deberán respetar:

* Roles globales
* Roles por módulo
* Permisos específicos

Ejemplo:

* Técnicos pueden iniciar reuniones internas
* Admins pueden unirse o supervisar
* Usuarios finales solo pueden entrar si el ticket lo permite

---

# CONSIDERACIONES IMPORTANTES

La calificación:

* NO debe bloquear el cierre definitivo eternamente
* Debe tener expiración configurable
* Debe evitar abuso o spam
* Debe permitir análisis estadístico posterior

El diseño debe priorizar:

* Simplicidad
* Rapidez
* Bajo número de clics
* UX limpia
* Métricas reutilizables para analítica futura
